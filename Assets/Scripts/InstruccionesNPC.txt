# Configuración del NPC de Misiones

## Scripts Creados:
1. **SistemaMisiones.cs** - Sistema central de misiones
2. **NPCMisiones.cs** - Controlador del NPC con diálogo
3. **UIProgresoMision.cs** - UI del slider de progreso

## Configuración en Unity:

### 1. Crear el NPC:
- Crea un GameObject vacío llamado "NPCMisiones"
- Agrega un modelo 3D o sprite para el personaje
- Agrega un Collider (Box Collider o Sphere Collider) marcado como "Is Trigger"
- Agrega el script **NPCMisiones.cs**

### 2. Configurar el NPCMisiones.cs:
Asigna los siguientes elementos en el Inspector:

**Diálogos:**
- Dialogo Inicial: Array de strings con el diálogo inicial
- Dialogo Misión Activa: Array para cuando la misión está en progreso
- Dialogo Misión Completada: Array para cuando se completa
- Dialogo Recompensa Entregada: Array final

**UI (crear previamente):**
- Panel Dialogo: GameObject con el panel de diálogo
- Texto Dialogo: TextMeshProUGUI para mostrar el texto
- Boton Continuar: Button para continuar diálogo
- Boton Aceptar Misión: Button para aceptar
- Boton Rechazar Misión: Button para rechazar

**UI Slider Progreso:**
- Panel Progreso: GameObject con el slider
- Slider Progreso: Slider component
- Texto Progreso: TextMeshProUGUI para mostrar "X/5 combates"

### 3. Crear la UI del Slider:
- Crea un Canvas llamado "UIMisiones"
- Dentro crea un Panel "PanelProgresoMision"
- Agrega un Slider (llamado "SliderProgreso")
- Agrega un TextMeshProUGUI (llamado "TextoProgreso")
- Agrega el script **UIProgresoMision.cs** al panel

### 4. Configurar UIProgresoMision.cs:
Asigna:
- Slider Progreso: El slider component
- Texto Progreso: El texto de progreso
- Configura colores: rojo (inicio), amarillo (mitad), verde (final)

### 5. Configurar Jugador:
- Asegúrate que el jugador tenga el tag "Player"
- Agrega un Collider al jugador para la detección

## Funcionamiento:
1. El jugador se acerca al NPC y presiona E
2. Aparece diálogo inicial con opción de aceptar/rechazar
3. Al aceptar, se activa la misión y aparece el slider
4. Cada victoria/captura incrementa el contador
5. Al llegar a 5, el NPC ofrece recompensas
6. Recibe: 1 Ultrabola + 1 Charmander nivel 5

## Requisitos:
- El jugador debe tener tag "Player"
- CharmanderData debe estar en Resources/Porkemones/Charmander
- UI debe usar TextMeshProUGUI
- Los colliders deben estar configurados como Trigger

## Integración:
El sistema ya está integrado con:
- Sistema de combate (FuncTurnos.cs)
- Sistema de inventario (GestorDeBatalla.cs)
- Sistema de captura de Pokémon

## Notas:
- La misión solo se puede completar una vez
- El slider aparece automáticamente al aceptar la misión
- Las recompensas se entregan al hablar con el NPC después de completar
- El progreso se guarda en la sesión actual
